# КК23 — Система лояльности
## Техническое задание v1.1

---

## 1. Обзор проекта

Цель — внедрить собственную систему лояльности для кофейни КК23 без замены существующего оборудования (терминал торгового эквайринга Т-Банка). Система работает параллельно с кассой и не требует интеграции с ней.

| Параметр | Значение |
|---|---|
| Домен (первые итерации) | `bonus.sysctrl.ru` — используется на период разработки и тестирования |
| Домены (production) | `kk23.ru` / `кк23.рф` — целевые домены после релиза |
| Касса | Торговый эквайринг Т-Банка — физический терминал, оплата картой и QR по СБП |
| Принцип | Бонусная система живёт отдельно от кассы. Бариста вводит сумму чека вручную в панели после оплаты |

## 2. Механика баллов

### 2.1 Начисление

**Формула:** сумма чека × коэффициент уровня = начисленные баллы

**1 балл = 1 рубль при списании.**

| Уровень | Порог (сумма покупок) | Кэшбэк | Бонус |
|---|---|---|---|
| ☕ Гость | 0 ₽ | 5% | — |
| 🥐 Постоянный | 3 000 ₽ | 7% | Скидка в день рождения |
| ⭐ Свой | 10 000 ₽ | 10% | Бесплатный напиток раз в месяц |

- Порог — накопленная сумма всех покупок гостя (не баллы). Уровень не понижается.
- Пример: чек 400 ₽ на уровне «Гость» → начислено 20 баллов (400 × 0.05).

### 2.2 Списание

- **Скидка рублями:** 100 баллов = 100 ₽ скидки на текущий заказ
- **Бесплатная позиция:** накопил 250 баллов → забирает моти или напиток (на выбор)
- Гость выбирает способ списания сам, бариста исполняет в панели

### 2.3 Срок жизни баллов

Баллы сгорают автоматически если гость не совершал покупок **6 месяцев** подряд. За 2 недели до сгорания — автоматическое уведомление в Telegram. Для будущих каналов уведомление добавляется после их подключения.

## 3. Архитектура системы

### 3.1 Схема

```
[ Гость ]
    └── Telegram Bot ──► [ API-сервер KK23 ] ──► [ База данных ]
                              ▲
                       [ Панель бариста ]
```

Первая версия запускается через Telegram. Единый бэкенд хранит гостей, баллы, транзакции, PIN-коды и pending-запросы. MAX рассматривается как будущий дополнительный канал после MVP, без влияния на первый запуск.

### 3.2 Компоненты

| Компонент | Описание |
|---|---|
| API-сервер | Node.js (Fastify) или Python (FastAPI) — бизнес-логика, начисление/списание баллов, уровни |
| База данных | PostgreSQL — гости, баллы, транзакции, уровни, pending-запросы |
| Панель бариста | Веб-приложение под планшет — поиск гостя, начисление, списание |
| Telegram Bot | Регистрация, выдача PNG-карты, баланс, история, push-уведомления, подтверждение списания |
| Генератор карт | Создаёт PNG-карту гостя с брендом KK23, именем, PIN, балансом и уровнем |
| MAX Bot | Future-канал после MVP, базовые функции по той же логике |

## 4. Каналы для гостей

### 4.1 Верификация по номеру телефона

Номер телефона — единый идентификатор гостя. В MVP он подтверждается через Telegram: бот запрашивает контакт пользователя.

| Канал | Способ верификации | Стоимость |
|---|---|---|
| Telegram Bot | Telegram запрашивает номер у пользователя | Бесплатно |
| MAX Bot (после MVP) | Через платформу MAX | Бесплатно |
| Прямой сайт (опц.) | SMS или звонок-сброс | ~0.5–2 ₽/SMS |

### 4.2 Функции Telegram-бота в MVP

- Регистрация (имя, телефон, дата рождения по желанию)
- Выдача бонусной PNG-карты с балансом, уровнем и PIN
- Просмотр баланса баллов и текущего уровня текстом
- Персональный 4-значный PIN для идентификации у бариста
- История транзакций
- Push-уведомления: начисление баллов, подтверждение списания, угроза сгорания, бонусы уровня
- Повторная отправка обновлённой карты после начисления или списания

### 4.3 Приоритет разработки каналов

| # | Канал | Причина приоритета |
|---|---|---|
| 1 | Telegram Bot | Быстрее всего в разработке, популярен среди аудитории кофеен, закрывает регистрацию, карту и подтверждения |
| 2 | MAX Bot | Future-канал после MVP, дополнительный охват |
| 3 | Прямой сайт (опц.) | Резервный вариант для гостей без мессенджера, потребует SMS/звонок-сброс |

### 4.4 PNG-карта гостя

PNG-карта — основной визуальный носитель программы лояльности в Telegram. Гость открывает её в чате и показывает PIN бариста.

Карта содержит:

- бренд KK23 и визуальный стиль кофейни
- имя гостя
- персональный 4-значный PIN
- текущий баланс баллов
- текущий уровень
- дату последнего обновления

Карта генерируется:

- сразу после регистрации
- после каждого начисления баллов
- после каждого подтверждённого списания
- по команде «Показать карту»

В MVP карту можно генерировать заново на сервере при каждом изменении баланса. Хранить постоянный публичный URL необязательно: бот может отправлять свежий PNG-файл прямо в Telegram.

## 5. Панель бариста

Веб-приложение, оптимизированное под планшет. Открывается в браузере, не требует установки. Никакого дополнительного оборудования — только планшет, который уже есть на стойке.

### 5.1 Идентификация гостя

Каждый гость получает уникальный **4-значный PIN** при регистрации. PIN статичный, показывается на PNG-карте и используется баристой для быстрого поиска гостя.

| Способ | Скорость | Надёжность | Когда использовать |
|---|---|---|---|
| 4-значный PIN с PNG-карты (основной) | ~3 сек | Высокая | Всегда |
| Последние 4 цифры телефона | ~7 сек | Средняя | Гость забыл PIN |
| Полный номер телефона | ~10 сек | Высокая | Резервный вариант |

### 5.2 Сценарий — начисление баллов

| Шаг | Действие бариста | Что происходит в системе |
|---|---|---|
| 1 | Спрашивает про карту лояльности | — |
| 2 | Гость показывает PNG-карту или называет свой 4-значный PIN | Бариста вводит PIN в панели |
| 3 | Карточка гостя открылась | Имя, уровень, баланс баллов |
| 4 | Гость оплачивает на терминале Т-Банка | Обычная оплата, независимо от системы |
| 5 | Бариста вводит сумму чека, нажимает «Начислить» | Система считает баллы по уровню и зачисляет |
| 6 | Гостю приходит уведомление в Telegram | +20 баллов начислено, новый баланс |
| 7 | Бот отправляет обновлённую PNG-карту | На карте свежий баланс и тот же PIN |

### 5.3 Сценарий — списание баллов

Списание инициируется **ДО оплаты** на терминале. Требует подтверждения от самого гостя через Telegram.

| Шаг | Действие бариста | Что происходит в системе |
|---|---|---|
| 1 | Гость говорит что хочет потратить баллы | — |
| 2 | Бариста находит гостя по PIN | Карточка гостя с балансом |
| 3 | Бариста вводит сумму списания, нажимает «Списать» | Создаётся pending-запрос, статус: ожидание |
| 4 | Гостю приходит push в Telegram | «Списать 150 баллов? Скидка 150 ₽ — Да / Нет» |
| 5 | Гость нажимает «Да» на своём телефоне | Статус pending → confirmed, баллы списаны |
| 6 | Бариста видит подтверждение на планшете | Называет итоговую сумму гостю |
| 7 | Гость оплачивает на терминале Т-Банка | Оплата уже с учётом скидки |
| 8 | Бот отправляет обновлённую PNG-карту | На карте новый баланс |

> **Таймаут:** если гость не подтвердил списание в течение **2 минут** — запрос отменяется автоматически. Баллы не списываются.

### 5.4 Сценарий — регистрация нового гостя

| Шаг | Действие бариста | Что происходит в системе |
|---|---|---|
| 1 | Гость говорит что карты нет | — |
| 2 | Бариста предлагает открыть Telegram-бота | Гость запускает бота и отправляет контакт |
| 3 | Гость вводит имя и дату рождения по желанию | Система создаёт профиль и выдаёт 4-значный PIN |
| 4 | Бот отправляет приветствие и PNG-карту | На карте PIN, баланс 0, текущий уровень |
| 5 | Бариста сразу начисляет баллы за текущий заказ | Первые баллы зачислены, бот отправляет обновлённую карту |

### 5.5 Экраны панели

- **Главный** — поле ввода 4-значного PIN, кнопка «Новый гость»
- **Карточка гостя** — имя, уровень, баланс, кнопки «Начислить» и «Списать»
- **Начисление** — ввод суммы чека, превью баллов до подтверждения
- **Списание** — ввод суммы скидки, статус ожидания подтверждения от гостя
- **Журнал смены** — все операции за текущую смену
- **Регистрация** — инструкция для гостя открыть Telegram-бота, резервный ввод имени и телефона администратором

## 7. Безопасность

### 7.1 Идентификация гостя

- 4-значный PIN — только для поиска гостя и начисления баллов
- Начисление баллов по PIN безопасно — гость только получает, потерь нет
- Списание баллов требует push-подтверждения в Telegram с телефона самого гостя
- Даже зная чужой PIN — списать баллы без Telegram-подтверждения невозможно
- Регистрация в MVP проходит через Telegram с передачей номера телефона

### 7.2 Push-подтверждение списания

- Запрос живёт максимум 2 минуты, после — автоматическая отмена
- Одновременно может быть только один активный pending-запрос на гостя
- Гость видит точную сумму списания перед подтверждением
- После подтверждения — мгновенное уведомление с новым балансом и обновлённая PNG-карта

### 7.3 Панель бариста

- Авторизация по логину и паролю, сессия 8 часов (одна смена)
- Все операции логируются с ID бариста и точным временем
- Ручное изменение баланса только у администратора (владелец)
- При 3 неверных вводах PIN — блокировка поиска на 1 минуту

### 7.4 Администратор

- Отдельная роль с доступом к полной истории и ручной корректировке баллов
- Просмотр статистики: новые гости, оборот баллов, топ гостей по тратам
- Возможность заблокировать гостя или сбросить его PIN

## 8. Технологический стек

| MAX Bot | MAX Bot API | Future-канал после MVP |
| Хостинг | VPS (Timeweb, Selectel, Reg.ru) | Российские серверы, нет санкционных рисков |
